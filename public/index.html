Machine Payments Protocol Demo

Put an MPP paywall
in front of any origin.

This Cloudflare Worker turns protected routes into paid routes using the MPP `Payment` authentication flow, then upgrades successful buyers into short-lived cookie sessions for repeat access.

Built-in flow
GET /premium → 402 → Authorization: Payment → Payment-Receipt

The proxy uses `mppx` with the Tempo method by default, forwards paid traffic to your origin, and issues an `auth_token` cookie valid for one hour.

What changes

The proxy speaks native MPP: `WWW-Authenticate: Payment`, `Authorization: Payment`, and `Payment-Receipt` are first-class protocol pieces.

Why the cookie still exists

MPP handles the payment exchange; the JWT cookie makes repeat browser and agent requests cheap during the 1-hour access window.

Default method

This demo is wired to the Tempo method with a configurable token address and recipient wallet in `wrangler.jsonc`.

Request flow

01
Client hits a protected route

The Worker checks your route patterns before touching the origin.

02
Worker returns an MPP challenge

A `402 Payment Required` response includes `WWW-Authenticate: Payment`.

03
Client retries with proof of payment

The next request carries `Authorization: Payment`.

04
Origin response comes back with a receipt

The proxy forwards the request, adds `Payment-Receipt`, and sets a JWT cookie.
